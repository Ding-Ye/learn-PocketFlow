# learn-PocketFlow

Re-implementing [PocketFlow](https://github.com/The-Pocket/PocketFlow) in Go, one mechanism per chapter. ## Pedagogy

Each chapter:
1. **Builds on the previous one** ‚Äî but each session is a separate Go module with no cross-session imports. You can read s05 without reading s04; the diff is in the docs.
2. **Ships 5+ tests** demonstrating the new primitive's invariants.
3. **Quotes the upstream source** with permalink + line range so you can verify our Go port matches the Python semantics.
4. **Calls out deliberate divergences** ‚Äî Go can't overload `>>` or do multiple inheritance the way Python does, so we explain *why* the port shapes things differently.

Pedagogy borrowed from [`shareAI-lab/learn-claude-code`](https://github.com/shareAI-lab/learn-claude-code) ‚Äî mental model first, then code, then upstream cross-reference.
